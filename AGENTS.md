## Auth + collections (added this build â€” see `AUTH_SETUP.md`)

- **Auth.js v5** (`next-auth@5.0.0-beta.31`) with **Google + GitHub OAuth**, **JWT sessions**, **Neon Postgres + Drizzle** (`lib/db/`). Config in `auth.ts`; route handler `app/api/auth/[...nextauth]/route.ts`.
- **No `middleware.ts`/`proxy.ts`** (Next 16 renamed middleware). Auth is checked via `auth()` in `lib/auth/dal.ts`, called from the server actions in `lib/collections/actions.ts`.
- **The archive stays static.** Session is client-side via `SessionProvider` (`components/auth/Providers.tsx`) + `useSession`, so `/`, `/feed`, `/project/[slug]`, etc. remain SSG; only `/api/auth/*` is dynamic. Do NOT call `auth()` in the root layout (it would force dynamic rendering site-wide).
- **Collections** (`lib/collections.ts`, a session-aware `useSyncExternalStore`): logged-IN â†’ cloud-synced via server actions (optimistic + reconcile); logged-OUT â†’ localStorage, **read-only and gated** (a save routes to `/signup?next=<path>`). `components/auth/CollectionsBridge.tsx` drives the mode and runs a one-time localStorageâ†’account migration on first login. Saves use bookmark icons (hearts/stars were tried and retired).
- **Secrets:** `.env.local` (gitignored; currently placeholders + a real `DATABASE_URL` locally) and the same vars in Vercel. `lib/site-url.ts` resolves the canonical origin (prefers `NEXT_PUBLIC_SITE_URL`, else `VERCEL_PROJECT_PRODUCTION_URL`, so OG/sitemap auto-follow the custom domain). DB schema in `lib/db/schema.ts`; `npm run db:push`/`db:generate`.

## Taste rules (hard-won across many review rounds â€” do not violate)

1. **Curation bar:** the audience is a beginner at their first hackathon. Only "wow, I didn't think of that" projects that actually won big (top-3 overall finishes preferred over sponsor/API prizes). No opaque technical flexes, no AI-dashboard slop, no new hardware or deep-ML additions (existing fun-framed hardware like the CPR robot stays).
2. **Descriptions** (`data/blurbs.json` / `data/why-won.json`) must be crystal clear on the FIRST read: complete natural sentences, explained like you'd tell a friend. No fragments, no "X, but for Y" phrasing, no jargon.
3. **Design language:** dark graphite (pure black `#000` body under `data-v="harbor"`) + frosted cards, Helvetica bold/regular ONLY, monochrome Luma-style line icons in the UI, awards quantified as "1st out of 699 teams", numbers in white with muted labels. The **one brand-color exception is the gold pixel-croc logo mark** (`public/croc.png` transparent, `app/icon.png` favicon) â€” the user added it deliberately; it lives in the header, signup card, and OG image (it was tried in the onboarding hero and removed). The dark theme is `data-v="harbor"` rendered server-side on `<body>` â€” never apply theming via client JS (it reintroduces the cream flash on reload).
4. **Motion:** smooth between-page fade (`app/template.tsx`, opacity-only so it never breaks fixed overlays), cards rise in, images fade in (`ProjectImage` `.img-fade`), save toast (`components/Toaster.tsx` + `lib/toast.ts`). All gated by `prefers-reduced-motion` and `body.no-motion` in `app/globals.css`.
5. **Workflow:** the user iterates visually. Verify every UI change with a real browser screenshot (the `browse` skill) before reporting it done. Lint + build must be clean. Commit/push only when asked; pushing `main` deploys to prod.
