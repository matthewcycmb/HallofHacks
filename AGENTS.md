# Hall of Hacks — project brief

A dark, Luma-inspired archive of hand-picked winning hackathon projects (22 projects, 14 hackathons). GitHub: `matthewcycmb/untitled`, branch `main`.

**The product:** single "Harbor" view — card feed grouped by hackathon on the left (sticky group labels become floating pills while scrolling), sticky detail console on the right with a playable YouTube demo and a white "Save to collection" CTA. Header: "Hall *of* Hacks" logo (cream italic "of") left; Search (live-filters the feed via `?q=`), Categories (`/categories` page, filters via `?cat=`), and My collection right, all with monochrome line icons. Projects open as full pages at `/project/[slug]` — the popup modal was deliberately removed; do not bring it back. Saves use bookmark icons into localStorage collections (hearts and stars were tried and retired).

**Run it:** `npm run build && npm run start` — but ALWAYS kill any server on the port first, then build, then start. Building while the old server runs invalidates the hashed CSS/JS chunks and the site renders as unstyled HTML.

**Data — never hand-edit `data/projects.json`.** It is generated by the pipeline: `scripts/import-reading-list.mjs` → `rebuild-dataset.mjs` → `sanitize.mjs` → `extract-colors.mjs` + `enrich-images.mjs` → `final-cut.mjs`. The roster is `data/final-cut.json` (array order = display order, first 10 = featured); plain-English descriptions live in `data/blurbs.json`; per-event field sizes for award quantification in `data/event-sizes.json`. Details in `data/README.md`.

## Taste rules (hard-won across many review rounds — do not violate)

1. **Curation bar:** the audience is a beginner at their first hackathon. Only "wow, I didn't think of that" projects that actually won big (top-3 overall finishes preferred over sponsor/API prizes). No opaque technical flexes, no AI-dashboard slop, no new hardware or deep-ML additions (existing fun-framed hardware like the CPR robot stays).
2. **Descriptions** (`data/blurbs.json`) must be crystal clear on the FIRST read: complete natural sentences, explained like you'd tell a friend. No fragments, no "X, but for Y" phrasing, no jargon. Every new project gets a blurb in this style.
3. **Design language:** dark graphite + frosted cards, Helvetica bold/regular ONLY, monochrome Luma-style line icons (never colored emoji), awards quantified as "1st out of 699 teams", numbers in white with muted labels. The dark theme is `data-v="harbor"` rendered server-side on `<body>` in `app/layout.tsx` — never apply theming via client JS (it reintroduces the cream flash on reload).
4. **Workflow:** the user iterates visually. Verify every UI change with a real browser screenshot before reporting it done. Lint + build must be clean.
